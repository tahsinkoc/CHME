# Velocity Rules for Real Time Transaction Defense

## Scope
This document defines operational standards for velocity control strategy in the FinTech SaaS platform. The objective is to keep controls deterministic, auditable, and compatible with snapshot based recovery.

## Control Objectives
The control plane for real time rule engine must enforce window sizing, rule priority, and adaptive throttle policies. Every workflow should preserve event ordering, maintain clear owner attribution, and surface decision evidence for audits.

## Signals and Monitoring
Primary monitoring signals include: rapid card attempts, ip burst, repeated beneficiary changes. Secondary indicators include transaction retries, customer communication lag, and unresolved alert age. Daily control reviews must compare expected versus observed activity.

## Failure Modes and Recovery
Common failure modes include over blocking, under blocking, delayed rule propagation. Recovery starts with containment, then root cause isolation, then deterministic replay using snapshot aligned state. Every incident closes with a checklist that updates control logic and runbooks.
