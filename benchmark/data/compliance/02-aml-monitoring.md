# AML Monitoring Operations and Alert Handling

## Scope
This document defines operational standards for anti money laundering monitoring in the FinTech SaaS platform. The objective is to keep controls deterministic, auditable, and compatible with snapshot based recovery.

## Control Objectives
The control plane for transaction surveillance must enforce scenario tuning, alert triage, and escalation playbooks. Every workflow should preserve event ordering, maintain clear owner attribution, and surface decision evidence for audits.

## Signals and Monitoring
Primary monitoring signals include: round amount transfer, unusual corridor, structuring behavior. Secondary indicators include transaction retries, customer communication lag, and unresolved alert age. Daily control reviews must compare expected versus observed activity.

## Failure Modes and Recovery
Common failure modes include alert backlog, false positive fatigue, delayed suspicious activity review. Recovery starts with containment, then root cause isolation, then deterministic replay using snapshot aligned state. Every incident closes with a checklist that updates control logic and runbooks.
