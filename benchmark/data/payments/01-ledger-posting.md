# Ledger Posting Rules and Double Entry Discipline

## Scope
This document defines operational standards for ledger posting engine in the FinTech SaaS platform. The objective is to keep controls deterministic, auditable, and compatible with snapshot based recovery.

## Control Objectives
The control plane for double entry bookkeeping must enforce balanced journal checks, posting windows, and account mapping. Every workflow should preserve event ordering, maintain clear owner attribution, and surface decision evidence for audits.

## Signals and Monitoring
Primary monitoring signals include: unbalanced transaction, retry loop, reconciliation delta. Secondary indicators include transaction retries, customer communication lag, and unresolved alert age. Daily control reviews must compare expected versus observed activity.

## Failure Modes and Recovery
Common failure modes include double post, out of order posting, orphan reversal. Recovery starts with containment, then root cause isolation, then deterministic replay using snapshot aligned state. Every incident closes with a checklist that updates control logic and runbooks.
